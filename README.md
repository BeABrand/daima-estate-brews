# Daima Coffee Estate

Marketing site for **Daima Coffee Estate** — a working highland farm producing single-origin Kenyan coffee. Server-rendered React app deployed to Cloudflare Workers.

## Stack

| Layer | Choice |
|------|-------|
| Framework | [TanStack Start](https://tanstack.com/start) (file-based routing + SSR) |
| UI | React 19 + Tailwind CSS v4 + Radix UI primitives + shadcn/ui patterns |
| Bundler | Vite 7 (wrapped by `@lovable.dev/vite-tanstack-config`) |
| Runtime | Cloudflare Workers (via `@cloudflare/vite-plugin` + `wrangler`) |
| Forms / data | `react-hook-form` + `zod`, `@tanstack/react-query` |
| Build platform | [Lovable](https://lovable.dev) |

## Getting started

```bash
# Install
npm install                # or bun install (lockfile is bun.lock)

# Develop
npm run dev                # vite dev — http://localhost:8080 (or :8081 if 8080 busy)

# Build
npm run build              # production build, outputs to dist/
npm run build:dev          # development-mode build (sourcemaps, no minification)

# Preview a built bundle
npm run preview

# Lint / format
npm run lint
npm run format
```

## Brand & design tokens

The palette lives in `src/styles.css` as OKLCH custom properties and matches the favicon SVG hex values:

| Token | OKLCH | Hex equivalent | Usage |
|------|-------|----------------|-------|
| `--bean` | `oklch(0.22 0.04 45)` | `#3a2a1c` | Deep coffee — primary, footer bg, theme-color |
| `--cream` | `oklch(0.97 0.012 80)` | `#f8efe1` | Background, primary-foreground |
| `--cherry` | `oklch(0.55 0.18 30)` | `#c44a3a` | Accent — eyebrows, highlights |
| `--leaf` | `oklch(0.42 0.09 145)` | – | Reserved for future use |

Typography:
- **Display:** Fraunces (serif, variable)
- **Body:** Inter (sans, variable)

Both are loaded from Google Fonts via `<link>` in `__root.tsx` (with preconnect hints) — **never** add `@import url(...)` for fonts to `styles.css`, or Tailwind's compiled output will push the import past the spec-mandated position.

## SEO architecture

Centralised in two files:

- **`src/lib/site-config.ts`** — `SITE_URL` (override via `VITE_SITE_URL` env, default `https://daimacoffee.com`), brand constants, `absoluteUrl()` helper.
- **`src/lib/seo.ts`** — `buildSeoMeta({path, title, description?, image?, type?, noindex?})` and `buildSeoLinks({path})` produce a consistent meta + canonical set per route.

What every page emits:

- `<title>`, `<meta name="description">`, keywords, author, publisher, robots, googlebot
- `<link rel="canonical">` (absolute)
- Open Graph: `og:title`, `og:description`, `og:type`, `og:url`, `og:image`, `og:site_name`, `og:locale`
- Twitter card: `summary_large_image` with `twitter:site/title/description/image`
- `theme-color`, `format-detection`, `application-name`, `apple-mobile-web-app-title`

JSON-LD structured data:

| Where | Type |
|-------|------|
| Root layout (every page) | `Organization` (with PostalAddress + ContactPoint), `WebSite` |
| `/blog` | `Blog` with `blogPost: BlogPosting[]` |
| `/blog/:slug` | `Article` (with publisher logo ImageObject), `BreadcrumbList` |

Static SEO files in `public/`:
- `robots.txt` — `Allow: /`, disallow `/api/`, sitemap reference
- `sitemap.xml` — all 6 static routes + 5 blog posts with `<lastmod>` from each post's date

When adding a route, call `buildSeoMeta` + `buildSeoLinks` from the route's `head` callback and add a `<url>` entry to `sitemap.xml`.

## Deployment

The app builds for Cloudflare Workers via `@cloudflare/vite-plugin`. Worker entry is `src/server.ts` (declared in both `vite.config.ts` `tanstackStart.server.entry` and `wrangler.jsonc` `main`).

```bash
npm run build              # produces dist/
npx wrangler deploy        # ship to Cloudflare (requires wrangler auth)
```

Set `VITE_SITE_URL` (and any other `VITE_*` env vars) in your deploy environment. They are inlined into `import.meta.env` at build time by the Lovable vite wrapper.

## Working with this repo

- **Branches:** every fix/refactor lives on its own branch off the current head (never off `main`). Branch naming: `fix/*`, `feat/*`, `chore/*`, `docs/*`, etc.
- **Commits:** Conventional Commits style (`fix:`, `feat:`, `docs:`, `chore:`, `refactor:`, `perf:`, `ci:`). No external attribution lines.
- **`main` branch:** never `git checkout`/merge/push to `main`/`master`/`production` from a session.
- **Auto-generated files:** `src/routeTree.gen.ts` is rewritten by the TanStack router plugin on every dev/build — do not edit by hand.
- **Lovable build wrapper:** `@lovable.dev/vite-tanstack-config` aggregates 10+ vite plugins (tanstackStart, viteReact, tailwindcss, tsConfigPaths, cloudflare, componentTagger, env-define, alias, dedupe, error logger, sandbox detection). It is **required for the Lovable platform** — do not replace it.
- **Cross-session continuity:** every conversation should append to `documents/SESSION_RECOVERY.md` and update `documents/RESUME_INSTRUCTIONS.md` so a fresh agent can pick up the work.
